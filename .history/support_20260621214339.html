New File Detection is based on Google Drive item identity rather than folder position.

As a result, moving an existing file or folder to a different location within the same Google Drive structure will generally NOT cause it to be detected as a new item.

The following actions are typically detected as new:

  • Newly uploaded files
  • Newly created folders
  • Newly copied files

The following actions are generally NOT detected as new:

  • Moving an existing file to another folder
  • Moving an existing folder to another location
  • Renaming files
  • Renaming folders

This behavior is intentional and helps prevent existing items from being incorrectly flagged as new simply because their location changed.

04Google Platform Limitations

GDrive To Sheet operates as a Google Sheets and Google Apps Script based solution that relies on:

  • ⚡ Google Drive — File repository mapping structures.
  • ⚡ Google Sheets — Destination output table generation.
  • ⚡ Google Apps Script — Engine executing calculations.

Certain core limitations are structural parameters strictly imposed by Google Cloud platform policies, including API quotas, script execution time limits (e.g., execution time limits that vary depending on account type and Google policies), and corporate permission restrictions. These external limitations are outside the technical control of GDrive To Sheet.
